# 217. Contains Duplicate

Source: [Blind 75](https://leetcode.com/problem-list/oizxjoit/)

### Approach 1 — Brute Force

Compare every element against every other element.

- Time: `O(n²)`
- Space: `O(1)`

---

### Approach 2 — Sort First

Sort array, then scan adjacent elements.

- duplicates become neighbors
- compare `nums[i]` with `nums[i-1]`
- Time: `O(n log n)`
- Space:
    - `O(1)` if in-place sort allowed
    - depends on sorting implementation

---

### Approach 3 — Hash Set / Map (Optimal)

Track seen values using a map.

- if number already exists → duplicate found
- otherwise insert into map
- Time: `O(n)`
- Space: `O(n)`

```go
seen := map[int]bool{}
```
